Side note: Interruptions and questions welcome!

What is Pair Programming?

  • Two people working on the same problem
  • One driver, one navigator
  • Driver is at the controls, focusing at a low level
    • syntax
    • executing commands
  • Navigator works at a higher level
    • strategy
    • integration
    • thinking a few steps ahead
  • Who’s done pair programming before?

My Experience

  • Pairing full time since July 2010 at Pivotal Labs/New Context
    • on web & mobile apps, not state-of-the-art computer science
  • We pair on all production code
  • Complicated "tricky" code
  • Simple CSS fixes
  • Ops even

How Does it Actually Work?

  • 2 keyboards, 2 mice, 27" & 24" screens
  • Standard workstation image and tooling
  • Rotate pairs every day
  • No one has their own workstation
  • Everyone can and does use pretty much any machine
  • Email stations available for personal tasks
  • Remote pairing with Skype, iChat

Why?

  • Each developer is empowered to contribute
  • Quality code delivered faster & cheaper
    • ship, observe results, iterate
  • It's fun & satisfying
    • because of the people you're working with
    • beacuse they're better
    • because you hire them for their pairing skill
    • because you achieve stuff! together! every day!
  • Bonus: Great way to interview candidates

Empowered Team, Sustainable Project

  • Knowledge sharing by rotating pairs regularly (daily)
    • fluency in utilized technologies
    • value of practices (TDD, CI)
    • nurturing the inner generalist by pairing on new components
    • break down silos & bottlenecks
    • diversify skill set
  • Approaches to design, system architecture, etc
  • Project history
  • Learn fast!

Higher Code Quality

  • Always two viewpoints, backgrounds, experiences
    • deeply immersed
  • Who does code reviews? How effective are they at increasing code quality? What's the processing time from submission to completion?
  • Constant code review
    • turns potentially threatening code review into co-ownership
  • Bugs are often caught as they’re introduced
    • navigator thinks about what driver is typing in

Cost Savings

  • Dramatically decrease impact of interruptions (HUGE!)
    • both external & self-inflicted
    • one person handles interrupt, other stays focused
  • Clearer understanding of requirements
  • Focus on business value, not interesting engineering problems
    • pairs tends to keep themselves honest
  • Clear code, easier to maintain in the long run
  • Important issues thrashed out on the spot
    • naming: "What should we call this exception?"
    • clarity, simplicity

Challenges

  • Difficult to keep own hours or work from home
    • an adjustment for me
  • Being focused 8 hours per day can be draining
    • but is fun & fulfilling
    • it's good to decompress regularly (ping pong works great)
    • you get used to it
  • Research situations…

Difficult Pairs

  • Distracted people (iPad, phone, email)
    • feels like you’re soloing
  • No passion, heart is not in it
    • the person makes no progress in developing their skills
    • minimal contribution
  • Crowders

Summary

  • Pairing is awesome, highly recommended
  • Learning pace outstanding, I made huge progress
  • Can be addictive churning out high-quality product
